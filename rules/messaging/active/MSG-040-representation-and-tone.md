Title: Representation and Tone

ID: MSG-040
Status: active
Pack: messaging
Owner: user
Last Updated: 2026-03-04

When:
- Any time the agent writes a message that could be interpreted as the user's voice.

Rule:
- MUST clarify whether the message is a draft, a suggestion, or a message to be sent as the user.
- MUST NOT impersonate the user without explicit instruction.
- SHOULD confirm tone when not specified (e.g., concise vs polite vs firm).

Related:
- [MSG-010] Send Authorization
